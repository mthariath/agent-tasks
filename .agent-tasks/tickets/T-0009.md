Context

The repo is ready for a public beta, but the npm package boundary is still janky. `agenttasks` depends on `@agenttasks/core` via `workspace:*`, core is still private, there is no license, and packed tarballs currently include development-only files.

Outcome

The first public npm release is publishable as a clean beta without packaging surprises.

Acceptance Criteria

- `agenttasks` and `@agenttasks/core` are both publishable packages.
- Tarballs contain only intentional runtime files.
- The install/runtime contract is explicit, including Bun as a requirement for `agenttasks tui`.
- Release docs and smoke checks cover the beta release path.

Notes

This is a packaging and release-hygiene pass, not a feature pass.

Completed work

- Made `@agenttasks/core` public and publishable alongside `agenttasks`.
- Added MIT licensing and package metadata for both published packages.
- Restricted published tarballs to runtime files only.
- Added package READMEs and beta install/release docs.
- Added `scripts/smoke-release.sh` and root `pnpm release:smoke`.
- Added a clear Bun-required error path for `agenttasks tui`.

Verification

- `pnpm build`
- `pnpm typecheck`
- `pnpm test`
- `zsh scripts/smoke-release.sh`

Smoke results

- packed `@agenttasks/core` and `agenttasks` tarballs successfully
- confirmed tarballs exclude `src/`, `test/`, and `.turbo/`
- installed both tarballs into a clean temp project
- verified `agenttasks --help`, `init`, `validate`, `ready`, and `plan`
- verified `agenttasks tui` fails cleanly with an explicit Bun requirement when Bun is absent
