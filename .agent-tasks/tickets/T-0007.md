## Context

The default workflow skills and WORKFLOW.md still teach agents mostly about tickets, planning, and optional MCP usage. They do not yet reflect the new serve coordinator, project-local extensions, or hook-enforced execution behavior.

## Outcome

The default workflow pack accurately teaches agents how coordinator-backed execution, project-local extensions, and hook behavior fit into the workflow without turning the skills into extension-authoring docs.

## Acceptance Criteria

- templates/default/.agent-tasks/WORKFLOW.md includes concise coordinator/extensions guidance.
- The 4 default workflow skills mention serve/extensions/hooks where relevant.
- generated default content is regenerated from the updated source files.
- Installed/default workflow content remains aligned with the source-of-truth files.
- Tests cover the new workflow-skill wording and generated/install alignment.

## Notes

Keep the pack ticket-centric. Extensions are an execution-policy layer, not a new primary workflow.

- updated `working-tickets`, `creating-tickets`, `refining-tickets`, and `managing-status` to reflect coordinator-backed execution, extension policy, and hook-enforced behavior where relevant
- regenerated embedded default content from the source workflow files
- synced the local installed `.agents/skills/agent-tasks-default-workflow/` copies to match the updated source skills
- strengthened init/default-content tests to assert the workflow pack now mentions serve/extensions guidance
