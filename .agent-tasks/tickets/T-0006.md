## Context

`agent-tasks` now has a local execution coordinator, but no programmable extension surface for project-specific orchestration policy.

## Outcome

The coordinator can load project-local TypeScript extensions, run typed lifecycle hooks with durable run records, and expose minimal command registration without weakening core execution invariants.

## Acceptance Criteria

- `serve` loads project-local TypeScript extensions from `.agent-tasks/extensions/`.
- Extensions can register typed lifecycle handlers and explicit project-local commands.
- Blocking hooks can reject guarded actions with a structured reason.
- Hook runs are persisted under `.agent-tasks/.runtime/hooks/`.
- CLI/MCP/serve execution actions share the same coordinator path instead of duplicating hook logic.
- Init installs extension scaffolding/templates and docs explain the boundary.
- Tests cover loading, blocking, failure recording, and runtime isolation.

## Notes

Keep the extension API intentionally small. The coordinator stays authoritative for claims, readiness, transitions, and runtime state.

Implemented:
- added a shared coordinator runtime in `packages/cli/src/coordinator.ts` that loads project-local TypeScript extensions with `jiti`
- routed CLI lifecycle commands, MCP execution tools, and `serve` endpoints through the same coordinator path
- added blocking `before_*` hooks, observational `after_*` hooks, `review_requested`, and project-local command registration
- persisted hook runs under `.agent-tasks/.runtime/hooks/` and mark interrupted runs as `abandoned` on startup
- added `extensions` and `hook-runs` CLI inspection commands plus `serve` endpoints for extension status, hook runs, and extension commands
- extended `init` to scaffold `.agent-tasks/extensions/README.md`, `api.ts`, and example TypeScript extensions
- documented the extension and hook model in `docs/EXTENSIONS.md`, `docs/HOOKS.md`, `docs/EXECUTION_MODEL.md`, `docs/CLI.md`, and `README.md`
- covered init scaffolding, blocking hooks, failed post-hooks, and shared coordinator behavior in core/CLI tests
