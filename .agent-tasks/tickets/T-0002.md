## Context

Users expect `.agent-tasks/` changes from the CLI, TUI, or direct file edits to appear in the TUI immediately and without confusing selection jumps.

## Outcome

Realtime refresh behavior is reliable, easy to trust, and clearly communicated in the shell.

## Acceptance Criteria

- Ticket and epic file changes appear promptly in an open TUI session.
- Selection is preserved when possible across reloads.
- Watcher state remains visible and understandable when reloads or watcher errors happen.
- Live behavior is covered by tests and a manual smoke path.

## Notes

Focus on correctness first, then UI feedback polish.
