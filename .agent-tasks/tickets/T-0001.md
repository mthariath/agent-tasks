## Context

The board currently collapses to a single visible status column earlier than it should. That makes medium-width terminals feel unnecessarily constrained.

## Outcome

The board keeps multiple status columns visible until genuinely narrow widths by compacting headers, cards, and lane chrome first, then degrading through preview lanes before true single-column mode.

## Acceptance Criteria

- Common medium terminal widths still show multiple statuses at once.
- The board uses a progressive density ladder instead of a binary multi-column/single-column switch.
- Side panels collapse to overlays before the board gives up horizontal space.
- At intermediate widths, the selected lane stays full-width while neighboring lanes remain visible as narrow previews.
- Single-column mode is reserved for truly narrow widths.
- Column headers and ticket cards stay readable after the compaction pass.
- Responsive behavior is covered by state/model tests where appropriate.

## Notes

Prefer information density and scanability over extra chrome.

Suggested direction:
- keep 3+ columns longer by collapsing the inspector first and tightening lane padding/header text
- use abbreviated status labels and denser cards before removing columns
- at smaller widths, show one primary lane plus thin left/right preview lanes instead of immediately dropping to one column
- reserve true single-column mode for the smallest widths only

Implemented so far:
- added a `compact` layout mode between medium and narrow
- kept up to 4 visible lanes in compact widths and delayed true single-column mode to smaller widths
- added neighbor preview lanes with ticket context instead of dropping directly to one full lane
- tightened board cards and removed redundant per-card status text inside lane views
- flattened the header and inspector sections to reduce stacked boxes and padding
- added click targets for route tabs, status-strip items, side lists, lanes, and tickets
- replaced boxed header badges with inline watcher/issue/visible stats
- changed route tabs to plain text tabs instead of bordered mini-boxes
- flattened status summary chips in the header into inline colored text
- reduced inspector chrome by turning metadata badges into inline text and lighter section headings
- collapsed compact/narrow shell layers so small screens no longer get a second top-bar summary row
- simplified the narrow status strip into a single text line instead of stacked widget-like boxes
- removed the footer entirely on narrow screens when no modal or overlay is active
- removed the extra main-panel title bar on compact/narrow people/plan views where the top bar already carries the context
- removed the persistent side detail pane and switched ticket detail to a modal surface on all screen sizes
- recomposed ticket detail around planning/execution context instead of a plain field dump
- added contextual colored badges for workflow status, ready/blocked/active state, execution phase, assignee, priority, and kind
- added direct detail actions for status and assignment flows

Current finding:
- the worst vertical stacking regression is gone after explicitly verifying a live TUI render in tmux
- the header now reads as a single compact shell row plus one summary row, which is materially better than the previous stacked pill treatment
- on truly narrow widths the shell now gets down to a compressed top row plus a single status-strip line before the board
- the detail modal is materially better than the old side pane, but the board itself still has heavy lane and card chrome from all the box separators
- the remaining waste is mostly inside the board: lane boxes, card boxes, and preview-lane framing still feel heavier than they should at small widths
- the next cleanup pass should focus on lane/card density and panel headers rather than adding more shell elements
- live rendering verification needs to stay part of this ticket, not just JSX inspection and passing tests
