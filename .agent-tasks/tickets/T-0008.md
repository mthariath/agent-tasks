## Context

The repo now has a clearer execution/coordinator model, updated workflow skills, and extension docs. The remaining gap is the always-on guidance layer: the managed AGENTS block and top-level docs do not clearly explain the guidance stack or when agents should look at extension policy.

## Outcome

The always-on agent guidance stays short but teaches the right mental model: AGENTS.md for minimal contract, WORKFLOW.md for project workflow semantics, skills for workflows, and extensions/docs for targeted execution policy.

## Acceptance Criteria

- The managed AGENTS block mentions coordinator-backed execution and checking extension docs when automation or execution policy matters.
- Top-level docs describe the layered guidance model clearly and concisely.
- Tests assert the managed guidance stays aligned with the intended contract.
- The changes do not bloat AGENTS.md into a long architecture manual.

## Notes

Keep the always-on layer short. It should point, not teach everything inline.
